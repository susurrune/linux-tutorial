入门指南

在开始之前，让我们了解 Linux 命令行的基本概念和工作原理。

什么是 Shell？

Shell 是用户与 Linux 内核之间的接口。它解释用户输入的命令并将其传递给内核执行，然后将结果返回给用户。常见的 Shell 包括：

Bash

  • bash Bourne Again Shell，最常用
  • sh 原始 Bourne Shell

现代 Shell

  • zsh Z Shell，功能丰富
  • fish Friendly Interactive Shell
💡 提示

查看当前使用的 Shell：echo $SHELL

命令的基本结构

command [-options] [arguments] # 示例：使用 ls 命令 ls -la /home/user ↑ ↑ ↑ 命令 选项 参数

文件系统结构

Linux 采用层级目录结构，根目录 / 是所有文件和目录的起点：

/ # 根目录 ├── /bin # 基本命令二进制文件 ├── /boot # 启动加载器文件 ├── /dev # 设备文件 ├── /etc # 配置文件 ├── /home # 用户主目录 ├── /lib # 共享库 ├── /mnt # 挂载点 ├── /opt # 可选软件包 ├── /proc # 进程信息 ├── /tmp # 临时文件 ├── /usr # 用户程序 └── /var # 可变数据
Module 01

文件与目录操作

文件和目录是 Linux 系统的基础。掌握这些核心命令是命令行使用的起点。

pwd print working directory

显示当前工作目录的完整路径（绝对路径）。这是确认你当前所在位置的最基本命令。

# 基本用法 pwd /home/username/projects # 显示逻辑路径（解析所有符号链接） pwd -L /home/username/link-to-projects # 显示物理路径（不解析符号链接） pwd -P /home/username/projects
📝 注意

在脚本中使用 pwd 获取当前目录时，建议使用 $(pwd) 或环境变量 $PWD。

ls list directory contents

列出目录内容。这是使用频率最高的命令之一，掌握其各种选项可以大大提高效率。

选项全称说明
-llong长格式列出，显示权限、所有者、大小、时间
-aall显示所有文件，包括以点开头的隐藏文件
-hhuman人类可读的文件大小（KB, MB, GB）
-ttime按修改时间排序（最新的在前）
-rreverse反向排序
-Ssize按文件大小排序（最大的在前）
-Rrecursive递归列出子目录
-ddirectory列出目录本身而非其内容
--color启用彩色输出
# 最常用的组合：长格式、显示隐藏文件、人类可读 ls -lah drwxr-xr-x 5 user group 4.0K Jan 15 10:30 . drwxr-xr-x 12 user group 4.0K Jan 14 09:00 .. -rw-r--r-- 1 user group 220 Jan 15 09:15 .bashrc drwxr-xr-x 3 user group 4.0K Jan 15 10:30 Documents # 按时间排序（最新文件在最后） ls -ltr # 只显示目录 ls -d */
🎯 实用示例：查找最近修改的文件
ls -lt /var/log | head -10

显示 /var/log 目录中最近修改的 10 个文件

cd change directory

切换当前工作目录。这是导航文件系统的核心命令。

# 切换到指定目录（绝对路径） cd /home/user/Documents # 切换到用户主目录 cd ~ cd # 不带参数也回到主目录 # 切换到上级目录 cd .. # 切换到刚才所在的目录（快速来回切换） cd - # 使用环境变量 cd \$HOME/projects
💡 Tab 自动补全

输入目录名的前几个字母，按 Tab 键自动补全。按两次 Tab 显示所有匹配项。

touch create file / update timestamp

创建空文件或更新现有文件的时间戳。如果不存在则创建，如果存在则更新时间。

选项说明
-a只更新访问时间（atime）
-m只更新修改时间（mtime）
-t使用指定时间而非当前时间
-c不创建不存在的文件
# 创建单个空文件 touch newfile.txt # 创建多个文件 touch file1.txt file2.txt file3.txt # 批量创建文件 touch file{1..10}.txt # 创建 file1.txt 到 file10.txt # 更新文件时间为特定日期 touch -t 202401151200 file.txt
mkdir make directories

创建目录。使用 -p 选项可以创建多级嵌套目录。

# 创建单个目录 mkdir newdir # 递归创建多级目录（如果不存在则创建父目录） mkdir -p projects/web/app/src # 创建多个同级目录 mkdir dir1 dir2 dir3 # 创建目录并显示详细信息 mkdir -pv a/b/c mkdir: 已创建目录 'a' mkdir: 已创建目录 'a/b' mkdir: 已创建目录 'a/b/c'
cp copy files and directories

复制文件或目录。这是文件管理中最常用的操作之一。

选项说明
-r递归复制（用于目录）
-a归档模式，保留所有属性（同 -dR --preserve=all）
-v详细模式，显示复制过程
-i覆盖前询问确认
-u仅在源文件比目标新或目标不存在时复制
-n不覆盖已有文件
--preserve保留指定属性（模式、所有权、时间戳）
# 复制单个文件 cp source.txt dest.txt # 复制到目录 cp file.txt /home/user/Documents/ # 复制整个目录（必须加 -r） cp -r source_dir/ dest_dir/ # 归档复制（保留链接、权限、时间戳等） cp -a source/ backup/ # 安全复制（覆盖前询问） cp -i file.txt /important/ # 复制多个文件到目录 cp file1.txt file2.txt file3.txt dest_dir/
⚠️ 警告

复制目录时不要忘记 -r 选项，否则会报错。cp -r 会跟随符号链接，如果需要保留链接使用 -a。

mv move / rename files

移动或重命名文件和目录。mv 实际上是"重命名"操作，无论是否在同一文件系统。

# 重命名文件 mv old_name.txt new_name.txt # 移动文件到目录 mv file.txt Documents/ # 移动多个文件 mv file1.txt file2.txt dest_dir/ # 移动并覆盖（强制） mv -f source dest # 交互式移动（覆盖前询问） mv -i source dest # 不覆盖已有文件 mv -n source dest
💡 技巧

使用 mv 重命名时如果目标已存在，会被覆盖。使用 mv -i 或 mv -n 来避免意外覆盖。

rm remove files or directories

永久删除文件或目录。删除的文件不会进入回收站，无法恢复，使用时务必小心！

选项说明
-r / -R递归删除（用于目录）
-f强制删除，不提示，忽略不存在的文件
-i删除前逐一询问确认
-v显示详细删除过程
--preserve-root防止误删根目录（默认启用）
# 删除单个文件 rm file.txt # 删除多个文件 rm file1.txt file2.txt # 删除空目录 rmdir empty_dir/ rm -d empty_dir/ # 删除非空目录（危险操作！） rm -r directory/ # 强制递归删除（极度危险！） rm -rf directory/
⚠️ 警告：rm -rf

rm -rf / 会删除整个系统！现代系统有 --preserve-root 保护，但仍需极度小心。使用 rm -rf 前务必确认路径正确。

🛡️ 安全的删除习惯

▼
  1. 删除前先用 ls 确认要删除的内容：ls target* 然后 rm target*
  2. 使用绝对路径：rm /home/user/file.txt
  3. 使用 -i 选项交互式删除：rm -ri directory/
  4. 使用 trash-cli 等工具代替 rm，提供回收站功能
  5. 删除前备份重要数据
find search for files

在目录层次结构中搜索文件。这是 Linux 中最强大的文件搜索工具，支持复杂的条件组合。

表达式说明
-name按名称搜索（区分大小写）
-iname按名称搜索（不区分大小写）
-type按类型搜索：f(文件), d(目录), l(链接)
-size按大小搜索：+10M, -1k, 100c
-mtime按修改时间搜索：-7(7天内), +30(30天前)
-exec对找到的文件执行命令
-delete删除找到的文件
-maxdepth限制搜索深度
# 按名称查找 find . -name "*.txt" # 查找并忽略大小写 find . -iname "*config*" # 查找目录 find . -type d -name "node_modules" # 查找大于 100MB 的文件 find . -size +100M # 查找 7 天内修改的文件 find . -mtime -7 # 查找并执行命令（删除旧日志） find /var/log -name "*.log" -mtime +30 -exec rm {} \; # 查找空文件并删除 find . -type f -empty -delete
🎯 实用示例：批量重命名
# 将所有 .txt 文件转换为 .bak find . -name "*.txt" -exec mv {} {}.bak \;
locate find files by name

通过预建的数据库快速查找文件。比 find 更快，但数据库需要定期更新。

# 基本搜索（模糊匹配） locate filename # 更新数据库（需要 root） sudo updatedb # 查找时忽略大小写 locate -i filename # 只显示存在的文件（避免显示已删除但数据库未更新的文件） locate -e filename
📝 locate vs find

locate 搜索数据库，速度快但可能不包含最新文件；find 实时搜索文件系统，准确但更慢。

Module 02

权限与进程管理

理解 Unix 权限模型和进程控制是系统管理的基础。

chmod change file mode bits

修改文件或目录的权限。Unix 权限系统由三组权限（所有者、组、其他）和三种访问类型（读、写、执行）组成。

权限数字表示

权限数值说明
r (read)4读取文件内容或列出目录
w (write)2修改文件内容或创建/删除目录项
x (execute)1执行文件或进入目录

常用权限组合

文件权限

  • 644 rw-r--r-- 普通文件
  • 755 rwxr-xr-x 可执行文件
  • 600 rw------- 私密文件
  • 777 rwxrwxrwx 完全开放（慎用）

目录权限

  • 755 rwxr-xr-x 标准目录
  • 700 rwx------ 私有目录
  • 777 rwxrwxrwx 共享目录
# 数字模式设置权限 chmod 755 script.sh chmod 644 file.txt # 符号模式：添加权限 chmod u+x script.sh # 给所有者添加执行权限 chmod go-w file.txt # 移除组和其他人的写权限 chmod a+r document # 给所有人添加读权限 # 递归修改目录权限 chmod -R 755 project/ # 参考模式（将 file1 的权限复制给 file2） chmod --reference=file1 file2
📝 权限符号说明

u=用户(user), g=组(group), o=其他(other), a=所有(all)

+=添加, -=移除, ==设置

chown change file owner

修改文件或目录的所有者和所属组。通常需要 root 权限。

# 修改所有者（需要 sudo） sudo chown newuser file.txt # 同时修改所有者和组 sudo chown newuser:newgroup file.txt sudo chown newuser.newgroup file.txt # 只修改组 sudo chown :newgroup file.txt sudo chgrp newgroup file.txt # 等效命令 # 递归修改目录所有权 sudo chown -R user:group /var/www
🎯 常见场景：Web 服务器权限
# 将网站目录所有权给 web 服务器用户 sudo chown -R www-data:www-data /var/www/html sudo chmod -R 755 /var/www/html
ps process status

显示当前进程状态。ps 可以显示系统上运行的进程的快照。

# 显示当前用户的进程 ps # 显示所有进程的详细信息（BSD 风格） ps aux USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND root 1 0.0 0.1 169364 5904 ? Ss Jan14 0:03 /sbin/init # 标准语法 ps -ef # 显示所有进程的完整格式 ps -ejH # 显示进程树 # 查找特定进程 ps aux | grep nginx
字段说明
PID进程ID
PPID父进程ID
%CPUCPU 使用率
%MEM内存使用率
STAT进程状态（R运行, S睡眠, Z僵尸等）
VSZ虚拟内存大小
RSS常驻内存大小
TTY控制终端
top / htop process viewer

实时显示系统进程和资源使用情况。htop 是 top 的增强版，提供更友好的交互界面。

# 启动 top top # top 交互命令 q 退出 k 杀死进程（输入 PID） r 修改进程优先级（renice） Space 刷新显示 h 帮助 M 按内存排序 P 按 CPU 排序 T 按时间排序 u 按用户过滤
htop # 启动 htop F1/h 帮助 F2/S 设置 F3// 搜索进程 F4// 过滤进程 F5/t 树形视图 F6// 选择排序列 F7/[ 减小 nice 值 F8/] 增加 nice 值 F9/k 杀死进程 F10/q 退出
💡 安装 htop

大多数发行版需要单独安装：sudo apt install htop 或 sudo dnf install htop

kill / killall terminate processes

向进程发送信号以控制其行为。默认发送 TERM 信号请求进程终止。

常用信号

信号数值说明
SIGHUP1终端断开，常用于重新加载配置
SIGINT2中断（Ctrl+C）
SIGQUIT3退出（Ctrl+\）
SIGKILL9强制终止，无法被捕获
SIGTERM15默认信号，优雅终止
SIGCONT18继续暂停的进程
SIGSTOP19暂停进程（Ctrl+Z）
# 默认终止信号（优雅终止） kill PID kill -15 PID kill -TERM PID # 强制终止（无法被捕获，最后手段） kill -9 PID kill -KILL PID # 按名称终止所有匹配进程 killall firefox killall -9 process_name # 重新加载配置（不终止进程） kill -HUP nginx_pid # 查看所有可用信号 kill -l
⚠️ SIGKILL vs SIGTERM

优先使用 SIGTERM（默认），给进程清理资源的机会。SIGKILL 是最后手段，进程无法捕获或忽略它，可能导致数据丢失。

Module 03

网络诊断

掌握网络诊断工具是系统管理员和开发者的必备技能。

ip network configuration

现代 Linux 网络配置工具，替代了 ifconfig。用于查看和配置网络接口、路由、策略路由等。

# 显示所有网络接口 ip addr ip a # 简写 # 简洁彩色输出 ip -br -c addr lo UNKNOWN 127.0.0.1/8 ::1/128 eth0 UP 192.168.1.100/24 # 显示特定接口 ip addr show eth0 # 显示路由表 ip route ip r # 显示链路层信息 ip link show
ping ICMP echo request

测试与目标主机的网络连通性。发送 ICMP echo 请求并等待回应。

# 基本用法（按 Ctrl+C 停止） ping google.com # 指定次数 ping -c 4 google.com # 指定间隔（秒） ping -i 2 google.com # 指定超时 ping -W 3 host # 洪水模式（root 权限，测试网络压力） sudo ping -f localhost
🎯 解读 ping 输出
64 bytes from 142.250.80.46: icmp_seq=1 ttl=117 time=15.3 ms ↑ ↑ ↑ ↑ ↑ 数据包大小 目标 IP 序列号 生存时间 响应时间
curl / wget transfer data from URLs

命令行下的网络数据传输工具。curl 功能更丰富，wget 更适合递归下载。

# 获取网页内容 curl https://api.github.com # 保存到文件 curl -o output.html https://example.com curl -O https://example.com/file.zip # 保留原文件名 # 跟随重定向 curl -L https://bit.ly/xxx # POST 请求 curl -X POST -d 'key=value' https://api.example.com # 添加 Header curl -H 'Authorization: Bearer token' https://api.example.com # 下载并显示进度 curl -# -o large.zip https://example.com/large.zip
# 下载文件 wget https://example.com/file.zip # 指定输出文件名 wget -O newname.zip https://example.com/file.zip # 后台下载 wget -b https://example.com/large.iso # 断点续传 wget -c https://example.com/large.iso # 递归下载网站 wget -r -l 2 https://example.com # 限制下载速度 wget --limit-rate=200k https://example.com/file
📝 curl vs wget

curl：功能全面，支持更多协议，适合 API 调用和复杂请求。
wget：擅长递归下载和断点续传，适合镜像网站和大文件下载。

Module 04

文本处理

Linux 文本处理三剑客：grep、sed、awk。掌握它们，你就能处理任何文本数据。

grep global regular expression print

搜索文本中的匹配行。支持基本和扩展正则表达式，是文本过滤的首选工具。

选项说明
-i忽略大小写
-v反向匹配（显示不匹配的行）
-n显示行号
-c计数匹配行
-l只显示包含匹配的文件名
-r递归搜索
-E使用扩展正则表达式（同 egrep）
-F固定字符串搜索（同 fgrep）
-w匹配整个单词
-o只输出匹配的部分
-A n显示匹配行及后 n 行
-B n显示匹配行及前 n 行
-C n显示匹配行及前后各 n 行
# 基本搜索 grep "error" log.txt # 忽略大小写 grep -i "warning" file.log # 显示行号 grep -n "pattern" file.txt # 递归搜索 grep -r "function" src/ # 扩展正则表达式 grep -E "error|warning|fatal" log.txt # 排除匹配行 grep -v "^#" config.txt # 排除注释行 # 显示上下文 grep -C 3 "Exception" app.log # 只输出匹配部分 grep -o "[0-9]\+\.[0-9]\+\.[0-9]\+" version.txt
🎯 实用示例：日志分析
# 统计 ERROR 出现次数 grep -c "ERROR" app.log # 查找不包含特定字符串的文件 grep -L "TODO" *.py # 复杂条件搜索 grep -E "(ERROR|FATAL)" log.txt | grep -v "DEBUG"
sed stream editor

流编辑器，用于对文本进行过滤和转换。常用于替换、删除、插入等操作。

# 基本替换：s/old/new/flags sed 's/foo/bar/' file.txt # 每行第一个匹配 sed 's/foo/bar/g' file.txt # 全局替换 sed 's/foo/bar/2' file.txt # 替换第2个匹配 sed 's/foo/bar/2g' file.txt # 从第2个开始替换 # 直接修改文件（-i） sed -i 's/foo/bar/g' file.txt sed -i.bak 's/foo/bar/g' file.txt # 创建备份 # 删除行 sed '1d' file.txt # 删除第1行 sed '1,5d' file.txt # 删除1-5行 sed '/^#/d' file.txt # 删除以#开头的行 # 打印特定行 sed -n '5,10p' file.txt # 只打印5-10行 sed -n '10p' file.txt # 只打印第10行 # 在指定行后添加内容 sed '5a\This is new line' file.txt # 多重命令 sed -e 's/foo/bar/' -e 's/baz/qux/' file.txt
💡 sed 定界符

可以使用任意字符作为定界符，当模式包含/时很有用：sed 's#/usr/local#/opt#g'

awk pattern scanning and processing

强大的文本分析工具，支持编程逻辑。适合处理结构化数据和生成报告。

# 打印特定列（默认空格分隔） awk '{print $1}' file.txt # 第1列 awk '{print $NF}' file.txt # 最后一列 awk '{print $2, $4}' file.txt # 第2和第4列 # 指定分隔符 awk -F: '{print $1}' /etc/passwd awk -F, '{print $2}' data.csv # 条件过滤 awk '$3 > 100 {print $0}' data.txt awk '/pattern/ {print}' file.txt # 计算总和 awk '{sum += $3} END {print sum}' data.txt awk '{sum += $3} END {print "Total:", sum}' data.txt # 计算平均值 awk '{sum += $3; count++} END {print sum/count}' data.txt # 统计行数 awk 'END {print NR}' file.txt # 复杂报告 awk -F: '{print "User:", $1, "\tShell:", $NF}' /etc/passwd
🎯 实用示例：进程内存统计
ps aux | awk '{sum += $6} END {printf "Total RSS: %.2f MB\n", sum/1024}'

正则表达式

正则表达式是文本处理的瑞士军刀，掌握它能让你的命令行能力倍增。

基本元字符

  • . 任意单个字符
  • * 前一字符0次或多次
  • + 前一字符1次或多次
  • ? 前一字符0次或1次
  • ^ 行首
  • $ 行尾
  • [] 字符集
  • [^] 否定字符集

字符类

  • \d 数字 [0-9]
  • \w 单词字符 [a-zA-Z0-9_]
  • \s 空白字符
  • \D 非数字
  • \W 非单词字符
  • \S 非空白字符

量词

  • {n} 恰好n次
  • {n,} 至少n次
  • {n,m} n到m次
  • *? 非贪婪*
  • +? 非贪婪+

分组与引用

  • () 捕获组
  • (?:) 非捕获组
  • | 或
  • \1 引用第1组
  • \2 引用第2组

练习 3：系统监控脚本

编写一个系统状态监控脚本。

#!/bin/bash # system_monitor.sh LOG_FILE="/var/log/system_monitor.log" echo "=== System Report $(date) ===" | tee -a $LOG_FILE # CPU 使用率 echo "CPU Usage:" top -bn1 | grep "Cpu(s)" | sed 's/.*, *\([0-9.]*\)%* id.*/\1/' | awk '{print 100 - $1"%"}' # 内存使用 echo "Memory Usage:" free -h | awk '/^Mem:/ {print $3"/"$2 " (" $3/$2 * 100.0 "%)"}' # 磁盘使用 echo "Disk Usage:" df -h | awk 'NR>1 {print $5 " " $1}' # 负载最高的进程 echo "Top CPU Processes:" ps aux | sort -nrk 3,3 | head -5 | awk '{print $11 " " $3"%"}'
