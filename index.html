文件与目录

📁
Essential

文件系统导航

Linux 文件系统是一棵倒置的树。理解路径是掌握命令行的第一步。

# 打印当前工作目录 pwd /home/user/projects # 列出文件详情 (-l 长格式, -a 显示隐藏文件) ls -la drwxr-xr-x 5 user group 4096 Jan 15 10:30 . -rw-r--r-- 1 user group 220 Jan 15 09:15 .bashrc # 切换目录 (. 当前目录, .. 上级目录, ~ 家目录) cd ~/Documents cd ..
✏️
Basic

文件操作

创建、复制、移动、删除——CRUD 的 shell 版本。

# 创建空文件或更新时间戳 touch file.txt # 创建目录 (-p 递归创建) mkdir -p projects/web/app # 复制 (-r 递归复制目录) cp -r source/ dest/ # 移动或重命名 mv old.txt new.txt # 删除 (-r 递归, -f 强制) rm -rf folder/

Tips

路径技巧

  • Tab×2 — 列出所有可能
  • Ctrl+C — 终止当前命令
  • Ctrl+L — 清屏
  • ↑/↓ — 历史命令
Module 02

权限与进程

Unix 安全模型的核心：读(4)、写(2)、执行(1)。

# 查看权限 ls -l -rwxr-xr-x 1 user group script.sh # 数字模式: 755 = rwxr-xr-x chmod 755 script.sh # 符号模式 chmod u+x script.sh chmod go-w file.txt # 修改所有者 sudo chown user:group file

每个程序都是一个进程。学会查看、监控和终止它们。

# 查看进程 (BSD风格) ps aux USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND root 1 0.0 0.1 169364 5904 ? Ss Jan14 0:03 /sbin/init # 实时进程监控 (q退出) top htop # 更友好的界面 # 查找进程 pgrep firefox ps aux | grep nginx # 终止进程 kill -9 PID killall firefox

进程间通信的基础。

  • SIGINT (2) — Ctrl+C 中断
  • SIGTERM (15) — 优雅终止
  • SIGKILL (9) — 强制杀死
  • SIGHUP (1) — 终端断开
Module 03

网络诊断

网络问题的诊断始于理解连接。从 IP 地址到数据包，掌握网络的每一层。

# IP 地址和接口 ip addr ip -br -c addr # 简洁彩色输出 # 路由表 ip route
# 连通性测试 ping -c 4 google.com traceroute google.com # DNS 查询 dig google.com nslookup google.com
# 端口和连接 ss -tuln # 查看监听端口 netstat -tuln # 下载工具 curl -O url wget url
Module 04

文本与工具

grep 搜索、sed 流编辑、awk 文本分析——Unix 哲学的完美体现。

# grep: 文本搜索 grep "error" log.txt # 基础搜索 grep -i "warning" file # 忽略大小写 grep -r "pattern" /var/log # 递归搜索 grep -E "error|warning" log # 正则表达式 # sed: 流编辑器 sed 's/old/new/g' file.txt # 全局替换 sed -i '1d' file # 删除第一行 sed -n '5,10p' file # 打印 5-10 行 # awk: 文本分析 awk '{print $1}' file # 打印第一列 awk -F: '{print $NF}' /etc/passwd awk 'NR>1 {sum+=$3} END {print sum}' data

不同发行版的软件管理。

# Debian/Ubuntu apt update apt install nginx apt remove nginx # CentOS/RHEL/Fedora dnf install nginx # Arch Linux pacman -S nginx

小工具组合的强大力量。

cat log.txt | grep "ERROR" | wc -l
